## Dataset Strategy

ATT uses three tiers of data, each serving a different validation purpose. Deep research confirmed that sliding-window PH on bistable perception EEG has NOT been published, making Tier 2 a genuinely open niche.

## Tier 1: Synthetic Systems (Built-in)

No downloads needed. Generated by `att.synthetic`. All generators accept a `seed` parameter for deterministic reproducibility.

| System | Topology | Use Case |
|--------|----------|----------|
| Lorenz | 2 dominant H1 loops (butterfly wings) | Ground truth for embedding + persistence |
| Rössler | 1 dominant H1 loop, band structure | Topological contrast with Lorenz |
| Coupled Lorenz | Tunable binding (coupling 0→1) | Binding detection + benchmark validation |
| Coupled Rössler-Lorenz | Heterogeneous timescales + tunable coupling | Per-channel delay validation |
| Switching Rössler | Known transition times | Transition detection validation |
| Coupled oscillators | N-body emergent topology | Multi-system binding |

These are the primary validation layer. Every algorithm must work here first. The coupled Lorenz sweep with benchmarks (binding score vs TE vs PAC vs CRQA) is itself a publishable figure.

## Tier 2: Bistable Perception EEG (Primary Neural Target)

### Why Bistable Perception

Bistable perception (binocular rivalry, Necker cube) is the cleanest natural example of attractor switching. The stimulus doesn't change — the brain spontaneously transitions between two perceptual interpretations. This is chaotic itinerancy made visible.

**Research gap confirmed**: No published work applies sliding-window persistent homology to EEG during binocular rivalry or Necker cube tasks. TDA on EEG has been applied to other paradigms (Gestalt grouping: Li et al.; working memory: Huang, Jin et al.) but not bistable perception.

### Primary Target: Nie, Katyal & Engel — Binocular Rivalry SSVEP

Nie et al., "An Accumulating Neural Signal Underlying Binocular Rivalry Dynamics" (J. Neuroscience 43(50):8777, 2023)
- 34-channel EEG (10/20 system), 84 subjects
- Continuous raw EEG with button-press perceptual switch markers ("tilt left", "tilt right", "mixed")
- SSVEP frequency tagging at 14.4 Hz and 18.0 Hz — objective percept dominance measure
- 12 runs × 120 sec per subject
- Hosted at UMN Data Repository: https://conservancy.umn.edu/handle/11299/257166
- DOI: 10.13020/9sy5-a716
- License: CC BY-NC 3.0 US
- Same lab as Katyal et al. (2015) — natural lineage from the earlier simultaneous EEG-fMRI work

**Actual downloaded specs** (differ from paper's raw recording parameters):
- **Sampling rate**: 360 Hz (preprocessed), NOT 1024 Hz raw
- **Format**: MATLAB .mat files (`scipy.io.loadmat` compatible), NOT BDF/EDF
- **Preprocessing already applied**: CSD transform, artifact rejection, ICA component removal, bandpass filtered
- **Subject 1 path**: `data/eeg/rivalry_ssvep/Sucharit - 012516_3629/`
- **Structure**: `Epochs/` (6 .mat files — rivalry + SFM + fixation + rest) and `Behavior/` (button-press .mat files)
- **Epoch shape**: (34, ~43195) float64 per rivalry session (~120s at 360 Hz)
- **Rivalry sessions**: 12 total (6 in `riv_12`, 6 in `riv_13` — two counterbalanced SSVEP frequency conditions)
- **Button presses**: `responseKey` field (1=tilt left, 2=tilt right, 3=mixed)

### Alternative Datasets

**Fallback: Resting-State Eyes-Open/Closed**

- Not bistable, but clear alpha-power state transition. Less novel but data guaranteed.

### Pre-Screening Protocol (Phase 3, Task 3.1-3.2)

Before committing to a dataset, verify:
1. Event markers exist and correctly label perceptual switches
2. Continuous raw EEG is available (not just epoched data)
3. At least 32 channels with standard 10-20 montage positions
4. At least 3 subjects with clean data
5. Sufficient number of switch events per subject (>20) for statistical power
6. File format readable by MNE-Python or scipy (BDF, EDF, SET, FIF, .mat)

### Preprocessing Pipeline

**For Nie/Katyal/Engel data**: The downloaded .mat files are already preprocessed (CSD, ICA, artifact rejection, bandpass). No MNE preprocessing needed. Load directly with `scipy.io.loadmat` and extract epoch arrays for embedding.

**For other datasets** (standard pipeline via MNE-Python):
1. Load raw files
2. Set montage (standard 10-20 or dataset-specific)
3. Bandpass filter: 1-45 Hz
4. Notch filter: 50/60 Hz (depending on recording location)
5. Re-reference to average
6. ICA artifact rejection (eye blinks, muscle artifacts)
7. Epoch around perceptual switch events: -2s to +2s
8. Extract channel time series for embedding

### Channel Selection Strategy

Run the full pipeline on each region independently, then use binding detection for cross-region coupling:

- **Parietal (P3, Pz, P4)**: Strongly implicated in perceptual bistability
- **Occipital (O1, Oz, O2)**: Primary visual processing
- **Frontal (F3, Fz, F4)**: Top-down influence on perception

For binding detection, primary pairs:
- Occipital ↔ Parietal (feedforward + feedback in visual hierarchy)
- Frontal ↔ Parietal (top-down modulation)

### EEG Embedding Strategy

**The problem**: AMI/FNN estimation on raw EEG is frequently unreliable due to noise, non-stationarity, and volume conduction. Debugging parameter estimation and neural topology simultaneously is a recipe for wasted time.

**The workflow** (automated via `EEGLoader.embed_channel()`):

1. **Try auto-estimation first**: `TakensEmbedder("auto", "auto")` on preprocessed, band-filtered EEG
2. **Check quality**: `validate_embedding()` on the result
3. **If degenerate**: Log warning with reason (e.g., "AMI no minimum", "condition > 1e4"), re-embed using fallback parameters
4. **Return**: Point cloud + metadata dict recording which method was used

In batch mode (multiple channels × subjects), this runs per-channel without exceptions. The metadata audit trail shows which channels used auto vs fallback after the batch completes.

**Fallback parameters** (accessible via `EEGLoader.get_fallback_params()`):

| Band | Bandpass (Hz) | Delay τ (samples @ 256 Hz) | Delay (ms) | Dimension d | Source |
|------|--------------|---------------------------|------------|-------------|--------|
| Broadband | (1, 45) | 10 | ≈39 ms | 5 | Stam (2005), standard EEG nonlinear analysis |
| Alpha | (8, 13) | 8 | ≈31 ms | 4 | Narrowband, lower dimension sufficient |
| Theta-Alpha | (4, 13) | 12 | ≈47 ms | 5 | Primary band for bistable perception |
| Gamma | (30, 45) | 3 | ≈12 ms | 5 | Fast dynamics, short delay needed |

**Rationale**: These are not arbitrary. The delay values correspond approximately to the first AMI minimum for band-filtered EEG signals at 256 Hz sampling rate, as reported in Stam (2005) "Nonlinear dynamical analysis of EEG and MEG" and corroborated by Lehnertz & Elger (1998). The dimension values follow FNN convergence for these bandwidths. Using published defaults lets us focus on the novel analysis (sliding-window PH, binding detection) rather than getting stuck on parameter estimation.

**When auto-estimation works well**: Clean EEG with strong oscillatory structure (e.g., alpha-band from occipital channels with eyes closed). AMI shows a clear first minimum and FNN converges cleanly.

**When auto-estimation fails**: Noisy broadband EEG, channels with strong muscle artifact, very short epochs (<1000 samples). AMI may be monotonically decreasing (no minimum) or FNN may not converge below threshold.

### Analysis Plan (1-Subject Proof-of-Concept)

The EEG phase is scoped to a single-subject proof-of-concept. The goal is to determine whether the pipeline produces meaningful results on real data, not to establish a neuroscience finding. If the 1-subject result is positive, scaling to 3-5 subjects is straightforward and deferred to Phase 5.

1. **Single-channel sliding-window PH**: Track topology evolution on occipital and parietal channels around switch events. Hypothesis: bottleneck distance spikes at or before button press.
2. **Cross-region binding**: Compare parietal-occipital binding score during stable percepts vs transition periods. Hypothesis: binding score changes during transitions.
3. **If positive**: Report as preliminary result in preprint with appropriate caveats (N=1, needs replication).
4. **If negative**: Document failure mode (embedding quality, condition numbers, signal characteristics) and report honestly in preprint discussion.

## Tier 3: Transformer Hidden States (Extension — Phase 5)

Not required for core project. For post-launch exploration.

### Approach

Extract hidden states from a transformer during processing. Treat the token sequence as a time axis, hidden dimensions as the state space. Apply Takens embedding to PCA-reduced trajectory. Compare topology across layers.

```python
from transformers import AutoModel, AutoTokenizer
import torch

model = AutoModel.from_pretrained("gpt2", output_hidden_states=True)
tokenizer = AutoTokenizer.from_pretrained("gpt2")

inputs = tokenizer("The cat sat on the mat", return_tensors="pt")
with torch.no_grad():
    outputs = model(**inputs)

# outputs.hidden_states: tuple of (n_layers+1) tensors
# Each: (batch, seq_len, hidden_dim)
```

### Potential Questions

- Does attractor topology change across layers?
- Do correct vs incorrect predictions differ topologically?
- Does model size affect attractor complexity?

## Ethical Notes

## Data Availability Risks

| Risk | Mitigation |
|------|-----------|
| Nie/Katyal/Engel data unavailable | Downloaded and verified. Subject 1 confirmed working. |
| Preprocessed data missing raw info | Data is at 360 Hz post-CSD/ICA — no raw recovery possible, but sufficient for embedding. |
| All rivalry datasets unusable | Fall back to resting state eyes open/closed. Less novel but guaranteed availability. |
| Datasets too large for local development | Subsample to 1-3 subjects. Use only EEG modality. |
| AMI/FNN fails on noisy EEG | Use fallback parameters from literature. See EEG Embedding Strategy above. |
